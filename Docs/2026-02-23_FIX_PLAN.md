1. **Hint placement (all labs with skeleton + inlineHints):** Run `npm test -- --run src/test/labs/validate-hint-rendering.test.ts`; fix every reported enhancement (line + blankText). Then do a visual check in the browser for "?" marker alignment.

Fix malformed keyConcepts (partial-recovery-overview, reporting-overview) and add more to auto-ha-overview.
3. **Prerequisites:** Add concrete prerequisites where empty (e.g. lab-rich-query-basics).
4. **Terminal / bash blocks:** Audit every enhancement file that has `filename: 'Terminal'` or `language: 'bash'`. Remove only blocks that **only** run `node file.cjs`/`node file.js`; replace with "Run all / Run selection" in instructions and tips. Document or keep blocks that run AWS CLI, mongosh, or multi-terminal workflows.
5. **Enhancement tips:** Ensure 2–4 tips per enhancement; mention "Run all or Run selection" where execution is from the editor.
6. **Labs without enhancements:** For lab-graph-*, lab-geospatial-*, lab-operations-monitoring, lab-data-change-streams: either add POV enhancements + loader prefix + step enhancementIds, or explicitly mark as "content-only / UI-only" and add keyConcepts + sourceProof/sourceSection.
7. **Step metadata:** Add sourceProof, sourceSection, and (where applicable) verificationId to steps that are missing them (graph, geospatial, monitoring, change-capture).
